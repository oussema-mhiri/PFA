1. PRÉSENTATION DU PROJET

🎯 Contexte et Problématique

ISITCOM forme chaque année plus de 1 800 étudiants. Problèmes :

  • 📚 Cours dispersés sur Moodle
  • 🎭 Clubs manquant de visibilité
  • 📅 Événements annoncés tardivement
  • 📄 Anciens examens et PFE difficiles à trouver
  • 📱 Pas d'interface mobile adaptée

🎯 Objectifs du projet

  • ✅ Plateforme web full-stack centralisée
  • ✅ Présentation des 3 licences et cycle ingénieur
  • ✅ Valorisation des clubs étudiants
  • ✅ Accès facile aux ressources pédagogiques
  • ✅ Calendrier centralisé des événements
  • ✅ Méthodologie Agile - équipe de 6 personnes

6. CAS D'UTILISATION

📌 UC-01 : S'authentifier

Acteurs : Étudiant, Responsable Club, Administrateur

Préconditions : Avoir un compte valide

Scénario nominal : 1. L'utilisateur accède à la page de connexion → 2. Il saisit email et mot de passe → 3. Le système vérifie les identifiants → 4. Redirection vers le dashboard adapté

📌 UC-02 : S'inscrire à un club

Acteurs : Étudiant

Préconditions : Authentifié et non déjà membre

Scénario nominal : 1. Consultation de la page du club → 2. Clic sur "Rejoindre" → 3. Formulaire d'inscription → 4. Validation → 5. Notification au responsable club

📌 UC-03 : Publier un événement

Acteurs : Responsable Club, Administrateur

Préconditions : Authentifié avec droits de publication

Scénario nominal : 1. Accès au dashboard → 2. "Ajouter un événement" → 3. Remplir les champs → 4. Validation → 5. Événement visible dans le calendrier

📌 UC-04 : Télécharger une ressource pédagogique

Acteurs : Étudiant

Préconditions : Authentifié

Scénario nominal : 1. Navigation dans "Ressources" → 2. Application des filtres → 3. Clic sur le document → 4. Téléchargement du fichier

9. CONCLUSION

✅ Réalisations prévues

  • Site web full-stack complet, responsive
  • 20 user stories implémentées sur 4 sprints
  • Architecture moderne (React + Laravel + MySQL)
